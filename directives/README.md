# Directives

SOPs en Markdown. Cada archivo describe un proceso: objetivo, inputs, herramientas requeridas, pasos y edge cases.

## Plantilla

```markdown
# <nombre del proceso>

## Objetivo
Qué se busca lograr.

## Inputs
Qué información o archivos necesita el proceso.

## Herramientas
Scripts en `execution/` o APIs externas que usa.

## Pasos
1. ...
2. ...

## Edge cases
- Qué hacer si falta un input
- Qué hacer si una API falla
```
